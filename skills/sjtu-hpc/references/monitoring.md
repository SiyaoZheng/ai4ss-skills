# Cluster Monitoring / 集群监控

**监控仪表板**: https://mon.hpc.sjtu.edu.cn/d/Q6W7OfInk/ (无需登录，查看节点状态和 Slurm 状态)

## 关键指标 (用于故障排查)

| 指标 | 正常值 | 异常时的影响 |
|------|--------|--------------|
| 登录节点状态 (sylogin/pilogin) | **On** | Off = 无法 SSH 登录 |
| 传输节点状态 (data/sydata) | **On** | Off = 无法传输数据 |
| Slurm 状态 (sacct/sinfo) | **On** | Off = 作业调度系统故障 |
| CPU 使用率 | <90% | >95% = 排队时间长 |
| GPU 使用率 | <90% | >95% = GPU 作业需要长时间等待 |

## 故障排查优先级

```
作业失败时，先检查集群状态，不要自己内耗找代码 bug！

1. 检查登录节点是否 On → 否则是集群问题
2. 检查 Slurm 状态是否 On → 否则是调度系统问题
3. 检查 CPU/GPU 使用率 → 过高说明资源紧张
4. 以上都正常 → 再检查自己的代码/脚本
```
