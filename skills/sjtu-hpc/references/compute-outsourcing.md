# 计算外包工作流 | Compute Outsourcing Workflow

对于计算社会科学研究项目（R/Python + targets/Make），推荐采用**本地协调 + HPC 计算**的混合模式：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    本地协调 + HPC 计算 工作流                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   本地机器 (Local)                        HPC 集群 (Remote)               │
│   ┌─────────────────┐                    ┌─────────────────┐            │
│   │ 项目协调/编排    │                    │ 计算密集型任务   │            │
│   │ - targets/Make  │  ──── 外包 ────→   │ - 大规模回归    │            │
│   │ - 代码版本控制   │                    │ - ML 训练       │            │
│   │ - 结果整合       │  ←── 回收 ────    │ - 文本处理      │            │
│   │ - 可视化/报告    │                    │ - 蒙特卡洛模拟  │            │
│   └─────────────────┘                    └─────────────────┘            │
│                                                                         │
│   关键原则:                                                              │
│   1. 本地保持项目的"大脑" - 协调、版本控制、最终结果                       │
│   2. HPC 作为"算力" - 临时外包计算任务，完成后回收结果                     │
│   3. 使用 rsync 保持同步 - 增量传输，断点续传                             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 典型流程

1. 本地开发代码，准备全量数据
2. `hpc-sync push` 同步代码和数据到 HPC
3. 在 HPC 上 `sbatch` 或 `make` 或 `tar_make()` 运行计算
4. `hpc-sync pull` 回收 output/ 结果到本地
5. 本地整合结果、生成报告

## 关键设计：隔离原则

- `_targets/` 不同步 — 本地和 HPC 各自独立维护
- 避免元数据冲突，保证数据安全
- 最终结果通过 `output/` 目录交换

```bash
# 使用 hpc-sync (推荐)
cd ~/projects/my_research
hpc-sync init          # 首次配置
hpc-sync push          # 推送
hpc-sync pull          # 回收结果

# 或手动 rsync
rsync -avP --exclude='.git' --exclude='_targets/objects' \
    ./ user@data.hpc.sjtu.edu.cn:/dssg/home/acct-XXX/user/my_research/
```
