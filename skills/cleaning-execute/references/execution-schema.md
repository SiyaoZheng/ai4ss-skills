# Execution Schema (`cleaning-execute`)

This document specifies how `cleaning-execute` translates a `cleaning_contract`
block into a reproducible R script. The contract defines *what* to do; this
schema defines *how* to do it correctly.

DDI Lifecycle 3.3 governs all cleaning operations. Every operation produces
exactly one entry in `processing_events[]` (DDI CV: `CleaningOperation`).

---

## Operation Order (NORMATIVE)

The R script MUST apply operations in this exact order. Any deviation
silently corrupts data.

```
1.  Read raw data with haven           (preserves variable + value labels)
2.  Apply universe filter              (removes excluded rows)
3.  Apply missing_treatment            (sets specified codes to NA, per variable)
4.  Apply recode_map                   (substitutes values, on non-NA only)
5.  Apply reverse                      (flips scale, on non-NA only)
6.  Apply coerce_to                    (casts storage type)
7.  Apply harmonization                (code_alignment + na_alignment, if any)
8.  Apply rename_to                    (exposes post-rename analysis names)
9.  Build derived_variables            (evaluated on cleaned + harmonized frame)
10. Attach weight_assignment           (normalizes weight column)
11. Apply include=false drops          (removes excluded columns)
12. Select analysis_vars               (final column subset, if not null)
13. Write output CSV                   (with readr::write_csv)
```

### Why this order matters

- **Universe BEFORE missing_treatment**: excluded rows shouldn't influence
  missing-rate statistics for included observations.
- **Missing_treatment BEFORE recode_map**: if you recode first, valid codes
  like `9 = doctorate` would get mapped before you check whether `9 = refused`.
  This is the classic positive-missing trap (eval data: PGRS 2014 A3A).
- **Recode BEFORE reverse**: reverse computes `n + 1 - x` against the post-recode
  scale, not the original codes.
- **Coerce AFTER recode + reverse**: type casting happens on final values
  (e.g., a 1–5 ordinal cast to integer after reverse).
- **Harmonization AFTER coerce**: per-code alignment maps source codes to
  canonical codes (and source NA codes to canonical sentinels 99996–99999).
- **Rename BEFORE derive**: contract authors write derivation rules against
  post-rename analysis names, so `rename_to` must be visible before derivation.
- **Derive AFTER harmonization**: derivation rules reference the canonical-coded
  variables, not the source-coded ones.
- **Weight AFTER derive**: derived variables (e.g., post-stratification
  weights) may BE the weight.
- **Drop / select LAST**: final projection happens only after every variable
  needed by derivations or weights has been created.

## R Script Template

```r
# <stem>-cleaning.R
# Generated by cleaning-execute on <ISO8601 UTC date>
# Source contract: ddi-metadata.yaml
# Source data:    <study.data_source>
# Reproduces:     <stem>-clean.csv (sha256: <checksum>)

# ── Dependencies ───────────────────────────────────────────────────────────
library(haven)      # read_dta, read_sav with label preservation
library(dplyr)      # filter, mutate, recode, case_when, na_if, if_else
library(labelled)   # set_value_labels, val_labels (for canonical sentinels)
library(readr)      # write_csv

# ── 1. Read raw data ───────────────────────────────────────────────────────
df <- haven::read_<dta|sav>("<study.data_source>")

cat(sprintf("Read %d rows × %d cols from %s\n",
            nrow(df), ncol(df), "<study.data_source>"))

# ── 2. Universe filter ─────────────────────────────────────────────────────
# (skip this block if cleaning_contract.universe.condition is null)
n_before <- nrow(df)
df <- df |> dplyr::filter(<universe.condition>)
cat(sprintf("Universe filter: %d → %d rows (%d excluded)\n",
            n_before, nrow(df), n_before - nrow(df)))

# ── 3. Missing treatment ───────────────────────────────────────────────────
# One block per variable_contracts[] entry that has missing_treatment.
# Use dplyr::na_if() for single-code → NA.
# Use dplyr::case_when() for multi-code → NA or ranges.

df <- df |> dplyr::mutate(
  <var_name> = dplyr::if_else(
    <var_name> %in% c(<missing_codes>), NA_real_, as.numeric(<var_name>)
  ),
  # ... one row per variable
)

# ── 4. Recode_map ──────────────────────────────────────────────────────────
# One mutate() call per variable with recode_map.
# Use dplyr::recode with .default = NA_real_ for safety.

df <- df |> dplyr::mutate(
  <var_name> = dplyr::recode(
    as.numeric(<var_name>),
    `<old_val>` = <new_val>,
    # ... explicit mappings
    .default = NA_real_
  ),
)

# ── 5. Reverse scale ───────────────────────────────────────────────────────
# Compute n from variable.representation.codes — do NOT hardcode.

df <- df |> dplyr::mutate(
  <var_name> = (<n_max> + 1) - <var_name>,  # n_max from codes
)

# ── 6. Coerce types ────────────────────────────────────────────────────────
# Use as.integer / as.numeric / as.character / as.logical based on coerce_to.

df <- df |> dplyr::mutate(
  <var_name> = as.<integer|numeric|character|logical>(<var_name>),
)

# ── 7. Harmonization (code_alignment + na_alignment) ──────────────────────
# Skip if harmonization_declaration is empty.
# Order: na_alignment first (canonical sentinels), then code_alignment.

# 7a. NA alignment — source missing codes → canonical sentinels
df <- df |> dplyr::mutate(
  <var_name> = dplyr::case_when(
    <var_name> == <source_na_code_1> ~ <canonical_sentinel_1>,
    <var_name> == <source_na_code_2> ~ <canonical_sentinel_2>,
    TRUE ~ <var_name>
  ),
)

# 7b. Code alignment — source value codes → target codes
df <- df |> dplyr::mutate(
  <var_name> = dplyr::recode(
    <var_name>,
    `<source_code>` = <target_code>,
    # ... per-code mappings from code_alignment
    .default = <var_name>     # passthrough for already-canonical values
  ),
)

# 7c. Set canonical sentinels as labelled values (for downstream tools)
df <- df |> labelled::set_value_labels(
  <var_name> = c(
    "Refused" = 99997,
    "Don't know" = 99998,
    "Inapplicable" = 99999,
    "Missing data" = 99996
  ),
)

# ── 8. Rename columns ──────────────────────────────────────────────────────
# Apply rename_to before derived variables so derivation_rule expressions can
# use post-rename analysis names.

df <- df |> dplyr::rename(
  <new_name> = <old_name>,
)

# ── 9. Derived variables ───────────────────────────────────────────────────
# One mutate() per derived variable. Paste derivation_rule verbatim.

df <- df |> dplyr::mutate(
  <output_name> = <derivation_rule>,
)

# ── 10. Weight assignment ──────────────────────────────────────────────────
# Skip if weight_assignment is null.
# If normalize: true, scale weights so sum = nrow(df).

df <- df |> dplyr::mutate(
  weight = <weight_var_id> / mean(<weight_var_id>, na.rm = TRUE)
)

# ── 11. Drop excluded columns ──────────────────────────────────────────────

df <- df |> dplyr::select(
  -<col_with_include_false_1>,
  -<col_with_include_false_2>,
)

# ── 12. Select analysis_vars ──────────────────────────────────────────────
# Skip if analysis_vars is null.

df <- df |> dplyr::select(<col_1>, <col_2>, ...)

# ── 13. Write output ──────────────────────────────────────────────────────
readr::write_csv(df, "<stem>-clean.csv")
cat(sprintf("Wrote %d rows × %d cols to %s\n",
            nrow(df), ncol(df), "<stem>-clean.csv"))
```

## Edge Cases

### Empty universe filter result

If `cleaning_contract.universe.condition` excludes every row, the script
writes an empty CSV. **Do NOT silently produce an empty file** — surface
a warning. Likely the universe expression is wrong (e.g., `country == "CN"`
but the dataset uses ISO codes `156`).

### Missing required variable

If `cleaning_contract.universe.condition` references a variable not in
the dataset (e.g., due to a SAV/DTA label mismatch), R fails with
`object not found`. Surface the variable name to the user.

### Recode collision

If `recode_map` maps two distinct source codes to the same target
(e.g., `{1: 0, 2: 0, 3: 1, 4: 1, 5: 1}` for hukou recoding), this is
intentional collapse — proceed. If two `derived_variables[]` rules have
the same `output_name`, this is an error — stop.

### Weight normalization

If `weight_assignment.normalize: true`, scale weights so `sum(w) = nrow(df)`.
This preserves the count interpretation. If `normalize: false`, write
weights as-is.

### Harmonization without code_alignment

A `harmonization_declaration` entry can have variable-level fields only
(predicate, justification, confidence) without per-code alignment.
In that case, do NOT apply any code-level transformation — the contract
is asserting semantic equivalence with no value remapping needed.

### `coerce_to: factor`

R-specific. Convert with `factor(x, levels = sort(unique(x)))` to ensure
deterministic level order across runs.

## Reproducibility Guarantees

The generated R script must satisfy:

1. **Deterministic**: running `Rscript <stem>-cleaning.R` twice produces
   byte-identical CSV output (same row order, same column order, same
   values). No random seed dependencies. No `Sys.time()` calls.

2. **Self-contained**: only `library()` calls listed in the dependencies
   block. No `tidyverse` umbrella import. No `setwd()`. No environment
   variables.

3. **Auditable**: every operation in the contract has a corresponding
   block in the script with a header comment matching the operation
   step number.

4. **Idempotent**: re-running the script overwrites the output CSV but
   does not touch `ddi-metadata.yaml` or the raw data file.

The cleaning-execute skill verifies (1) by computing SHA256 of the
output CSV after a second run and comparing.

## CLI / Bash Interface

If the user wants to run cleaning-execute outside of Claude Code (e.g., in
CI), the skill can be invoked headlessly:

```bash
# Read contract, generate R script, execute, append audit event
python3 -m cleaning_execute \
  --metadata ddi-metadata.yaml \
  --output-prefix cgss_2021 \
  --strict          # fail if any _needs_review variable lacks a contract
```

The Python entry point reads the contract, writes the R script, runs it
with `subprocess.run(["Rscript", ...])`, captures the output, and appends
the `processing_events` entry. This is what the agent does internally —
exposing it as a CLI lets pipelines run without Claude.

(CLI implementation is optional; the SKILL.md workflow is the SSOT.)
