# Style Gates

Use this file when auditing a ggplot2 figure for top social-science journals. The style target is disciplined evidence, not visual novelty.

## Journal Anchors

AEA style requires figure files to be vector-based when possible and asks authors to label panel files clearly and place source notes after other notes related to the figure. APSA style emphasizes that figures should be interpretable without the text, use consistent graphics and typography, include keys for symbols or patterns, and remain viewable in black and white. ggplot2 supports PDF, SVG, EPS, and raster export through `ggsave()`; choose vector output for line, point, coefficient, event-study, and faceted plots.

## Required Gates

Pass all gates before calling a figure paper-ready:

- Style profile: every final figure uses the same declared style profile or records a substantive exception.
- Style source: repeated style choices live in a shared R style file or one shared style object, not scattered ad hoc `theme()` calls.
- Style consistency: typography, dimensions, aspect ratios, margins, axis formatting, point/line/interval geoms, palette, non-color encodings, legend placement, facet strips, panel tags, annotations, source-note convention, export device, and filename pattern are consistent across the manuscript's figure set.
- Vector-first export: PDF or SVG unless the object is a photo, screenshot, map tile, or other raster image.
- Single inquiry: the title, caption, and panels answer one empirical question.
- Honest uncertainty: confidence intervals, standard errors, bands, or explicit no-interval rationale are visible.
- Source note: source artifact, access or construction note, and sample period are stated.
- Sample note: unit, restrictions, dropped rows if relevant, weights, and clustering or interval rule are stated.
- Axis integrity: units, zero or transformed scale choices, binning, and axis truncation are explicit.
- Black-and-white readability: color is not the only carrier of meaning; use line type, point shape, labels, or patterns.
- Panel discipline: panel labels, scales, legends, and captions are consistent.
- Caption discipline: the caption describes what is plotted, not the manuscript's strongest interpretation.
- Claim boundary: causal or mechanism language matches methods-review status.
- Reproducibility: script path, output path, and commands are recoverable.
- ggplot2 durability: the script creates named plot objects and saves them with explicit `ggsave()` width, height, units, and device or extension.
- Tool transparency: helper-package output from `fixest`, `ggfixest`, `did`, `marginaleffects`, `modelsummary`, `cregg`, `binsreg`, `rdrobust`, `sf`, `ggdist`, or similar tools is traceable to declared plotted data and not treated as a black box.

## Reject Or Repair

Reject or repair figures with any of these problems:

- synthetic, simulated, illustrative, generated, benchmark-calibrated, or random-draw empirical rows
- axis choices that hide sign, magnitude, uncertainty, or subgroup differences
- smoothing or binning not declared in code or notes
- legends or colors that fail in grayscale
- unsupported "effect" language for descriptive plots
- caption claims not supported by the plotted quantity
- raster-only exports for line, point, coefficient, or event-study plots
- `ggsave()` calls that depend on the last displayed plot for final manuscript figures
- final paper figures generated by Python, Stata, JavaScript, an IDE export button, screenshots, dashboards, or interactive widgets instead of reproducible R/ggplot2 code
- helper-package defaults that hide the reference category, omitted event time, binning, bandwidth, projection, transformation, interval rule, or sample restriction
- inconsistent figure styles across the same manuscript, including changing fonts, widths, legend positions, line widths, point sizes, interval styles, palettes, direct-label rules, panel tags, or export devices without a recorded substantive reason

## Default Aesthetic

Use quiet typography, direct labels where helpful, light gridlines, no heavy fills, no decorative gradients, and enough whitespace for journal column width. Avoid oversized titles inside figures; the paper caption carries the title. Prefer `theme_classic()` or a restrained `theme_minimal()` with larger axis text and explicit legend placement.
