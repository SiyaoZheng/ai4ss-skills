# Design System — ai4ss Companion HTML Reports

> Design brief for the three self-contained Python-generated HTML report viewers.
> Core goal: the report should communicate "The AI did good work" at first glance —
> quality gate confidence, not generic data display.

---

## Visual Direction: "Audit Panel"

The aesthetic of a trusted instrument. VS Code diff view meets a clean data audit
dashboard. Every visual element earns its place by carrying signal; no decorative
chrome, no marketing gradients, no serif warmth.

Comparators studied: Datasette (research data tool), Observable (notebook paradigm).
Key insight: confidence comes from information density + clean hierarchy + honest
error surfacing, not from decoration.

---

## Typography

| Role | Font | Size | Weight |
|------|------|------|--------|
| UI labels, prose | `system-ui, -apple-system, Inter, sans-serif` | 13–15px | 400/500 |
| Variable names, column headers | `JetBrains Mono, monospace` | 12–13px | 500 |
| Data values, code, LaTeX | `JetBrains Mono, monospace` | 12px | 400 |
| Section headings | `system-ui` | 18–22px | 600 |
| Report title | `system-ui` | 28px | 700 |

JetBrains Mono is embedded as a base64 WOFF2 subset (Latin + common symbols only)
so reports stay fully self-contained.

## Color System

```css
:root {
  --bg:           #ffffff;
  --surface:      #f8f9fa;
  --border:       #e2e8f0;
  --text:         #0f172a;
  --text-muted:   #64748b;
  --accent:       #2563eb;   /* primary — variable links, selected state, focus rings */
  --warn:         #f59e0b;   /* ⚠️ positive-missing traps */
  --ok:           #22c55e;   /* clean classification */
  --critical:     #ef4444;   /* missing code errors, structural N/A flags */
  --code-bg:      #f1f5f9;   /* inline code blocks */

```

No dark mode — reports are designed for review in a bright office/lab context.
No gradients. No shadows beyond `box-shadow: 0 1px 3px rgba(0,0,0,.06)` on cards.

## Component 1 — Codebook Browser (`codebook_html.py`)

**Input:** `ddi-metadata.yaml`
**Key UX goal:** PI can scan 600+ variables in <5 min, spot every ⚠️ trap, confirm AI did right work.

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│ STAT STRIP: 695 vars · 23 ⚠️ flagged · 2 🔴 critical · ... │  ← 64px strip
├──────────────┬──────────────────────────────────────────────┤
│              │                                              │
│  SIDEBAR     │  DETAIL PANEL                                │
│  280px fixed │  flex-grow                                   │
│              │                                              │
│  [search]    │  A3A — Highest education level               │
│              │  ────────────────────────────                │
│  A1 ✓        │  Type: categorical   Source: A3A             │
│  A2 ✓        │                                              │
│ ▶ A3A ⚠️     │  ┌─ ⚠️ POSITIVE-MISSING TRAP ──────────────┐│
│  A3B ✓       │  │ val_labels() shows 9 = doctorate (valid) ││
│  A4  🔴      │  │ NOT missing. Do not recode to NA.        ││
│  ...         │  └───────────────────────────────────────────┘│
│              │                                              │
│              │  Value Labels                                │
│              │  ┌────┬──────────────────┬──────────────────┐│
│              │  │ 1  │ No schooling     │ valid            ││
│              │  │ 2  │ Primary school   │ valid            ││
│              │  │ ...│ ...              │ ...              ││
│              │  │ 9  │ Doctorate        │ ✓ valid          ││
│              │  └────┴──────────────────┴──────────────────┘│
└──────────────┴──────────────────────────────────────────────┘
```

### Stat Strip
Large monospace counters, horizontal rule below. Example:
```
695          23           2            670
variables    ⚠️ flagged   🔴 critical   ✓ clean
```
Numbers in `--accent` (695, 670) or `--warn` (23) or `--critical` (2).

### Sidebar Variable List
- 100% height, sticky, overflow-y: scroll
- Each row: `[flag icon] [var name in JetBrains Mono]`
- Flag icons: `⚠️` warn, `🔴` critical, `✓` ok (muted green)
- Active row: `--accent` left border (3px solid), light blue bg `#eff6ff`
- Search input: full-width, `border: 1px solid var(--border)`, no box shadow

### Variable Detail Card
- Left border accent (4px): `--warn` for ⚠️, `--critical` for 🔴, `--ok` for clean
- Flag explanation box: `background: #fffbeb; border: 1px solid #fde68a` for warn
- Value labels table: alternating `--row-alt`, monospace values, classification badge

### Interactions (vanilla JS, no framework)
- Click sidebar row → update detail panel (no page reload — single HTML file)
- Search filters sidebar in real time
- Filter buttons: `All · ⚠️ Flagged · 🔴 Critical · ✓ Clean`

## Component 2 — LaTeX Table Viewer (`table_html.py`)

**Input:** `.tex` file (or raw LaTeX string from skill output)
**Key UX goal:** PI sees what the LaTeX looks like without compiling, confirms column alignment and stars.

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│ Table: table_1.tex · AEA style        [⇄ Split] [◀ Code] [▶ Preview] │
├───────────────────────────┬───────────────────────────────────┤
│  RAW LaTeX                │  RENDERED PREVIEW                 │
│  (syntax highlighted)     │  (CSS booktabs table)             │
│                           │                                   │
│  \begin{table}...         │  ┌────────────────────────────┐  │
│  \toprule                 │  │ Variable │ (1)  │ (2)       │  │
│  Variable & (1) & (2) \\  │  ├════════════════════════════┤  │
│  \midrule                 │  │ Age      │0.23* │ 0.18**    │  │
│  Age & 0.23* & 0.18** \\  │  │ Income   │...   │ ...       │  │
│  ...                      │  └────────────────────────────┘  │
└───────────────────────────┴───────────────────────────────────┘
│ Notes: Standard errors in parentheses. * p<0.1 ** p<0.05 *** p<0.01 │
└──────────────────────────────────────────────────────────────────────┘
```

### LaTeX Rendering
CSS-only booktabs simulation:
- `\toprule` → `border-top: 2px solid var(--text)`
- `\midrule` → `border-bottom: 1px solid var(--text); border-top: 1px solid var(--text)`
- `\bottomrule` → `border-bottom: 2px solid var(--text)`
- Significance stars rendered literally; decimal alignment via `text-align: right`

### Syntax Highlighting
Inline CSS highlight classes (no Prism/hljs dependency):
- LaTeX commands (`\begin`, `\toprule`) → `--accent`
- Numbers → `#0369a1`
- Stars → `--warn`
- Comments (`%`) → `--text-muted`

### Toggle
Three buttons (split / code-only / preview-only) switch layout class on container div.
No JS frameworks, no page reload.

## Component 3 — Performance Report (`perf_html.py`)

**Input:** before R code + after R code + `perf-summary.json` (optional)
**Key UX goal:** PI / HPC user sees exactly what changed and the speedup number prominently.

### Layout

```
┌─────────────────────────────────────────────────────────────┐
│ SPEEDUP BANNER: 12.4× faster · 87% memory reduction        │  ← 80px, prominent
├─────────────────────────────────────────────────────────────┤
│ METRIC CARDS                                                │
│ ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│ │ Runtime     │  │ Memory      │  │ Iterations  │         │
│ │ 4.2s → 0.34s│  │ 812MB→94MB  │  │ loops: O(n²)│         │
│ │ ████░░░░░░░ │  │ ████░░░░░░░ │  │  → vectorized│        │
│ └─────────────┘  └─────────────┘  └─────────────┘         │
├─────────────────────────────────────────────────────────────┤
│ CODE DIFF                                                   │
│ BEFORE                        │  AFTER                      │
│ ─────────────────────────     │  ──────────────────────     │
│  for(i in 1:nrow(df)) {       │  df$result <- sapply(       │
│    df$result[i] <- slow_fn()  │    df$x, fast_fn)           │
│  }                            │                             │
└─────────────────────────────────────────────────────────────┘
```

### Speedup Banner
- `background: linear-gradient(135deg, #eff6ff 0%, #dbeafe 100%)`
- Large speedup number (48px bold, `--accent`): `12.4×`
- Sub-label: `faster than baseline · bottleneck eliminated`

### Metric Cards
- 3-column grid, each card with before/after value and inline SVG bar chart
- Before bar: `--text-muted` fill; After bar: `--accent` fill
- Reduction percentage badge: `--ok` background

### Code Diff
- Side-by-side, line numbers in gutter
- Changed lines highlighted: removed lines `background: #fff1f2` (red tint), added `background: #f0fdf4` (green tint)
- No external diff library — Python `difflib` generates the HTML diff at report-build time

## Implementation Notes

### Self-Contained HTML
All CSS and JS inline in `<style>` and `<script>` tags. No CDN links. No external fonts
(JetBrains Mono as base64 WOFF2 subset). Reports open correctly with `python3 -m http.server`
or direct `file://` URL.

### Python Generator Pattern
Each generator follows `generate_pdf_report.py` pattern:
```python
def generate_html(input_path: str, output_path: str) -> None:
    data = load_input(input_path)
    html = render_template(data)
    Path(output_path).write_text(html, encoding='utf-8')
```

Template is an f-string (or `string.Template`) — no Jinja2 dependency.

### File Locations
```
codebook-parse.skill/scripts/codebook_html.py
latex-tables.skill/scripts/table_html.py
r-performance.skill/scripts/perf_html.py
```
