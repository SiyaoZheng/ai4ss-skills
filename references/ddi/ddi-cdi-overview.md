# DDI-CDI (DDI Cross-Domain Integration) v1.0

> Source: https://ddialliance.org/ddi-cdi
> Fetched: 2026-04-30

## Summary

DDI-CDI is a domain-neutral standard designed to describe data and process at the lowest level of granularity (the datum itself). It covers the majority of data structures in common use: Wide, Long, Multi-Dimensional and Key-Value. It offers a mechanism to interoperate disparate data from multiple disciplines and domains.

## Key Features

- **Model-driven**: Uses UML classes, expressed in Canonical XMI
- **Domain-independent**: Works with any research domain
- **Datum-oriented**: Describes data at the individual datum level
- **Provenance-focused**: Process description down to datapoint level if required

## Data Structures Supported

- Rectangular (wide)
- Long (event)
- Multi-dimensional (cube)
- No-SQL (big data)

## UML Model

Core is a UML model expressed in Canonical XMI (UML 2.5, XMI 2.5.1). Conforms to UML Class Model Interoperable Subset (UCMIS).

Model file: `ddi-cdi_canonical-unique-names.xmi`

## Supports Activities

1. Rich set of foundational metadata for variables, classifications, and other concepts and representations
2. Describes data in rectangular (wide), long (event), multi-dimensional (cube), and no-SQL (big data) data formats
3. Describes process as the primary aspect of data provenance
